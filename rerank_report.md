## When Does Re-Ranking Pay Off?

Re-ranking pays off when high-precision information retrieval is critical to downstream user satisfaction, and capturing deep semantic relevance outweighs raw speed. In our 60-pair labeled set, re-ranking surfaced gold documents for complex, long-tail technical queries where keyword overlap was minimal. For example, on the query *"How does the 0x55555555 mask work in the OpenJDK Integer.bitCount popcount implementation?"*, the hybrid baseline placed the gold document outside the top 5 due to heavy lexical competition from other bitwise operations. The Cross-Encoder, processing the query and candidate texts jointly, successfully evaluated the exact explanatory context and pushed the correct document to the top spot. 

## Latency Overhead

The cross-encoder introduces an overhead of 91.2 ms per query, which accounts for roughly 86% of the total execution time. This overhead scales linearly ($O(k)$) with `k_in` because the model must run an attention-heavy inference pass for every single candidate pair generated by Stage 1. However, this overhead remains perfectly flat and independent of the absolute corpus size, since the Cross-Encoder only observes the filtered down pool of 50 candidates. Conversely, the Stage 1 hybrid retrieval latency scales with corpus growth due to larger index lookups, shifting the relative performance ratio over time.

## At What Corpus Size or Query Volume Does It Stop Being Worth It?

The Cross-Encoder ceases to be viable once query volume crosses 10-11 Queries Per Second (QPS) on this CPU environment, as a single thread saturates at $1000\text{ ms} / 91.2\text{ ms} \approx 10.9\text{ QPS}$, causing severe request queuing. In terms of data scale, when the corpus expands past 250,000 documents, the Stage 1 hybrid retrieval latency naturally slows down. Once Stage 1 takes longer than ~40 ms, the cumulative multi-stage latency breaks strict sub-150ms SLAs, demanding a shift toward aggressive embedding caching or a learned, highly optimized vector-native re-ranker.
